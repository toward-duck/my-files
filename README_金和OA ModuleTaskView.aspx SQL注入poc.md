# 金和 OA C6 ModuleTaskView.aspx SQL注入漏洞自动化检测工具

本工具用于自动化检测 **金和企业管理软件（金和 OA / 金和 C6）** 协同任务模块（`ModuleTaskView.aspx`）中存在的**高危时间盲注（Time-based Blind SQL Injection）漏洞**。

为了解决传统时间盲注脚本在面对网络波动、死链或高延迟环境时的**高误报/漏报率**问题，本工具引入了**精准差分时间验证算法（双重请求时间差对比）**。通过对同一个目标发送不同延时权重的请求，计算响应时间差，实现工业级的漏洞验证。

---

## 🛠️ 漏洞说明

- **漏洞组件**：金和 C6 OA 系统（协同任务/日常任务管理模块）
- **漏洞接口**：`/c6/Jhsoft.Web.dailytaskmanage/ModuleTaskView.aspx`
- **注入参数**：POST 请求体中的 `OriginID`
- **漏洞类型**：SQL 注入（时间盲注 - Microsoft SQL Server 驱动）
- **风险等级**：**高危 (High)**。攻击者可通过该注入点脱取数据库敏感信息，甚至在特定配置下通过堆叠注入执行系统命令。

---

## ✨ 功能特性

- ⚡ **多进程并发扫描**：基于 Python `multiprocessing` 进程池，支持多线程并发，极大提高批量资产测绘效率。
- 🎯 **高精度差分判定**：
  - 分别向目标发送 `1秒基准延时` 和 `5秒目标延时` 请求。
  - 通过计算时间差值 $\Delta t = t_{5s} - t_{1s}$，只有当差值严格落在弹性网络波动区间（$2.5s \le \Delta t \le 5.5s$）时才判定存在漏洞。
- 🔍 **全状态实时输出**：
  - `[+]`：100% 确认存在 SQL 注入漏洞（符合差分延时特征）。
  - `[-]`：目标系统正常响应，未触发延时（系统安全）。
  - `[?]`：基准请求成功，但目标延时请求触发了 10 秒硬超时（极高概率存在漏洞，需手动复查）。
  - `[!]`：目标属于死链、断网或拒绝访问（不再无脑误报为漏洞）。
- 🔒 **进程安全日志**：利用 `multiprocessing.Lock` 进程锁，确保并发写入 `vuln.txt` 报告时数据不串行、不丢失。
